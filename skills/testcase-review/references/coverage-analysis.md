# 覆盖率分析与缺口识别指南

## 覆盖率计算方法

### 功能点覆盖率

```
功能点覆盖率 = (已覆盖的功能点数 / 总功能点数) × 100%
```

**功能点定义**：
- 需求文档中明确列出的功能项
- 每个用户可操作的功能单元
- 每个业务规则验证点

### 模块覆盖率

```
模块覆盖率 = (该模块已有用例数 / 该模块需求功能点数) × 100%
```

## 覆盖率等级标准

| 等级 | 覆盖率 | 说明 |
|------|--------|------|
| 优秀 | ≥ 90% | 全面覆盖，可上线 |
| 良好 | 70-89% | 基本覆盖，轻微风险 |
| 一般 | 50-69% | 部分覆盖，中等风险 |
| 不足 | < 50% | 大量缺失，高风险 |

## 缺口识别算法

### 1. 文本相似度匹配

```python
def match_functionality(case_title: str, requirement_text: str) -> float:
    """
    计算用例标题与需求描述的相似度

    Returns:
        0.0 - 1.0 的相似度分数
    """
    # 使用关键词重叠或语义相似度
    case_keywords = extract_keywords(case_title)
    req_keywords = extract_keywords(requirement_text)
    overlap = len(case_keywords & req_keywords)
    total = len(req_keywords)
    return overlap / total if total > 0 else 0
```

### 2. 模块级映射

```
需求模块 A
  ├── 功能点 A1
  ├── 功能点 A2
  └── 功能点 A3

用例模块 A
  ├── 用例 1 (覆盖 A1)
  ├── 用例 2 (覆盖 A2)
  └── 用例 3 (覆盖 A1, A2)

未覆盖：A3
```

### 3. 语义扩展识别

通过同义词、近义词扩展匹配：

| 需求关键词 | 用例可能表述 |
|-----------|-------------|
| 登录 | 登入、登录系统、用户登录 |
| 注册 | 账号注册、用户注册、注册账号 |
| 删除 | 移除、清除、销毁、删除数据 |
| 修改 | 编辑、更新、变更、修改信息 |

## 常见缺口模式

### 模式1：隐含需求未覆盖

**识别方法**：需求中的"应支持"、"应能"、"应当"等表述

```
需求：系统应支持大文件上传
用例：仅有小文件上传测试
→ 缺失：大文件上传测试
```

### 模式2：异常流程未覆盖

**识别方法**：正向流程有，异常流程无

```
正向：正常支付流程
缺失：
  - 支付中断处理
  - 支付超时重试
  - 支付失败提示
  - 重复支付防护
```

### 模式3：权限场景未覆盖

**识别方法**：有功能用例，无权限用例

```
功能：查看用户列表 ✓
权限：
  - 普通用户不能查看 ✗
  - 管理员可以查看 ✓
  - 非管理员访问被拒绝 ✗
```

### 模式4：兼容性场景未覆盖

**识别方法**：功能测试全，兼容性无

```
Web功能：登录、注册、发布 ✓
兼容性：
  - 浏览器兼容（Chrome/Firefox/Safari） ✗
  - 分辨率适配（1920/1366/1280） ✗
  - 移动端适配 ✗
```

## 优先级评估

### 影响度评估

| 影响度 | 说明 | 示例 |
|--------|------|------|
| 高 | 核心流程受阻 | 登录功能完全失效 |
| 中 | 功能部分受损 | 特定条件下功能异常 |
| 低 | 体验问题 | UI显示轻微偏差 |

### 紧急度评估

| 紧急度 | 说明 | 示例 |
|--------|------|------|
| 高 | 线上已发现缺陷 | 已有用户反馈 |
| 中 | 高概率触发 | 边界条件频繁使用 |
| 低 | 低概率触发 | 极端特殊场景 |

### 优先级矩阵

|  | 低影响 | 中影响 | 高影响 |
|--|-------|-------|-------|
| **高紧急** | P1 | P0 | P0 |
| **中紧急** | P2 | P1 | P0 |
| **低紧急** | P2 | P2 | P1 |
