# 测试用例评审方法论

## 评审目标

测试用例评审的核心目标是：
1. **完整性** - 确保需求中的每个功能点都有对应测试用例
2. **准确性** - 确保用例描述与需求一致
3. **有效性** - 确保用例能够有效发现缺陷

---

## 评审维度

### 1. 覆盖度评审

| 维度 | 说明 | 评审标准 |
|------|------|---------|
| 功能覆盖 | 需求中的功能点是否都有用例 | 覆盖率 ≥ 90% |
| 场景覆盖 | 主流程、分支流程、异常流程是否完整 | 主要场景无遗漏 |
| 数据覆盖 | 边界值、等价类是否覆盖 | 关键边界有用例 |

### 2. 用例质量评审

| 维度 | 说明 | 评审标准 |
|------|------|---------|
| 正确性 | 用例描述与需求一致 | 无歧义、无矛盾 |
| 可执行性 | 步骤清晰、可复现 | 每个步骤可操作 |
| 完整性 | 包含前置条件、步骤、预期结果 | 三要素完整 |
| 独立性 | 用例间无依赖 | 可独立执行 |

### 3. 优先级评审

| 优先级 | 定义 | 用例占比 |
|--------|------|---------|
| P0 | 核心功能、主流程 | 10-25% |
| P1 | 重要功能、主流场景 | 30-60% |
| P2 | 边缘功能、特殊场景 | 10-25% |

---

## 缺口识别类型

### 类型1：功能缺失

**识别特征**：
- 需求中明确描述的功能点
- 用例中完全没有对应测试

**示例**：
```
需求：支持第三方账号登录（微信、GitHub）
用例：仅有本地账号登录用例
→ 缺失：第三方登录功能测试
```

### 类型2：场景缺失

**识别特征**：
- 异常流程、分支流程无用例
- 边界条件未覆盖

**示例**：
```
需求：用户名长度为4-20字符
用例：仅测试"正常长度"场景
→ 缺失：边界值测试（3字符、21字符）
```

### 类型3：数据缺失

**识别特征**：
- 未覆盖空值、特殊字符
- 未覆盖最大/最小值

### 类型4：组合场景缺失

**识别特征**：
- 多功能组合操作无测试
- 跨模块业务流程无覆盖

## 评审流程

```
┌─────────────────────────────────────────────────────────┐
│  Step 1: 收集文档                                        │
│  - 测试用例Excel                                         │
│  - 需求文档（Word/PDF/PRD/图片）                          │
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│  Step 2: 解析结构                                        │
│  - 提取用例模块、标题、步骤                               │
│  - 提取需求功能点（按模块分类）                           │
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│  Step 3: 建立映射                                        │
│  - 用例功能点 → 需求功能点                               │
│  - 识别未映射的功能点                                     │
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│  Step 4: 缺口分析                                        │
│  - 分类缺口类型（功能/场景/数据/组合）                    │
│  - 评估影响和风险                                        │
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│  Step 5: 输出报告                                        │
│  - 覆盖率统计                                            │
│  - 缺失清单（含建议用例描述）                             │
│  - 补充优先级建议                                         │
└─────────────────────────────────────────────────────────┘
```

## 评审输出模板

### 覆盖率概览表

| 模块 | 需求功能点数 | 已有用例数 | 缺失用例数 | 覆盖率 |
|------|-------------|-----------|-----------|--------|
| 登录 | 8 | 6 | 2 | 75% |
| 注册 | 5 | 5 | 0 | 100% |
| 首页 | 12 | 8 | 4 | 67% |

### 缺失用例清单

#### 1. [模块名称] - 功能缺失

| 缺失功能点 | 需求描述 | 建议用例类型 | 建议优先级 |
|-----------|---------|-------------|-----------|
| 第三方登录 | 支持微信/GitHub登录 | 功能测试 | P1 |
| 登录失效处理 | Token过期需重新登录 | 异常测试 | P1 |

#### 2. [模块名称] - 边界场景缺失

| 缺失场景 | 边界条件 | 建议用例类型 | 建议优先级 |
|---------|---------|-------------|-----------|
| 用户名长度最小值 | 4字符以下 | 边界测试 | P2 |
| 用户名长度最大值 | 20字符以上 | 边界测试 | P2 |

## 关键评审检查点

### 登录模块检查点
- [ ] 正常登录成功
- [ ] 密码错误
- [ ] 账号不存在
- [ ] 账号被禁用
- [ ] Token过期处理
- [ ] 单设备登录冲突
- [ ] 第三方登录（微信/GitHub）
- [ ] 登录日志记录

### 注册模块检查点
- [ ] 正常注册成功
- [ ] 用户名已存在
- [ ] 密码格式不符合要求
- [ ] 邮箱格式错误
- [ ] 手机号格式错误
- [ ] 验证码错误/过期
- [ ] 重复提交处理

### 表单输入检查点
- [ ] 空输入
- [ ] 正常输入
- [ ] 边界值（最小/最大）
- [ ] 超长输入
- [ ] 特殊字符
- [ ] SQL注入/XSS攻击
- [ ] 空格/首尾空格处理

## 评审记录

每次评审建议记录：
- 评审日期
- 评审人
- 评审范围
- 发现问题数
- 补充用例建议数
- 实际补充数（后续跟踪）
