# 公众号爆款技术文章创作技能
> 适配标准：10%引入 + 10%原理 + 80%实操干货 黄金爆款结构
> 核心定位：彻底告别教科书式水文，只产出高收藏、高转发、高落地性的技术爆款推文

## ⚠️ 核心原则（硬性红线，不可违反）
1. **🎯 封面图强制要求** - 完整成文场景必须生成1张主题封面图；仅选题/标题需求不自动生成
2. **图片文字规范** - 所有生成图片上文字必须使用**标准简体中文**，无乱码、无扭曲
3. **图片数量管控** - 封面图1张（完整成文必出）+ 内容配图0-2张（非必要不添加）
4. **文章输出规范** - ❌ 禁止额外添加参考资料、附录等冗余章节，只输出正文主体
5. **链接格式规范** - 严格使用纯文本展示 ❌错误：`[官网](url)` ✅正确：`官网：url`
6. **按需执行原则** - 用户要哪步只做哪步；仅要选题只出选题、仅要标题只出标题，未要求完整成文时，不自动搜索、不写正文、不生成封面图
7. **平台合规原则** - 禁用限流敏感词，不出现绝对化夸大、引流私域、虚假承诺类表述
8. **干货比例铁律** - 全文严格执行 **10%引入 + 10%原理 + 80%实操干货** 配比，杜绝理论堆砌、水文凑字

## 第一步：热点选题
### 角色
资深软件测试专家 + 技术内容创作者 + 互联网行业分析师

### 输入
- {{热点领域}}（可选，默认兜底：AI/测试/自动化/大模型应用）

### 任务
输出当期最具爆款潜力 Top5 热点选题，每个选题附带完整维度评分，优先筛选**实操性强、可落地、受众广**的选题，规避纯理论、空洞概念类话题

### 高流量选题方向
| 类型 | 流量特点 | 示例 |
|------|----------|------|
| 薪资&职业发展 | 流量王者、转发极高 | "AI测试工程师薪资大揭秘" |
| 工具对比评测 | 实用性强、收藏率高 | "XX vs XX：AI测试工具哪家强？" |
| 实战案例复盘 | 代入感强、传播广 | "3天落地AI自动化测试，全过程复盘" |
| 入门科普教程 | 受众最广、新人必看 | "测试工程师入门AI，一篇吃透实操技巧" |
| 行业趋势洞察 | 树立专业人设 | "AI测试会逐步替代传统手工测试吗？" |

### 精简4维选题评估模型
| 维度 | 评估核心 | 分值权重 |
|------|----------|----------|
| 话题热度 | 全网搜索、微信指数、行业讨论度 | 25分 |
| 用户兴趣 | 目标受众规模、职场刚需程度 | 25分 |
| 干货价值 | 读者可学到实操知识点、能落地 | 25分 |
| 差异化 | 竞品发文少、视角新颖不烂大街 | 25分 |

总分：100分，70分以上为优质可写选题

### 选题否决红线
- 话题过于小众，目标受众小于1万人
- 同类竞品已发布10篇以上，内容严重同质化
- 热点时效性过期，热度降温超过2周
- 涉及政治、敏感行业、争议违规话题
- 无实操价值、纯空洞概念堆砌，无法落地

### 固定输出格式
📌 选题 [序号]：[选题完整标题]
🔥 热点来源：[行业趋势 / 产品新发 / 职场争议 / 新技术落地]
💡 为什么值得写：[1-2 句话精准概括价值]👥 目标读者：[测试工程师 / 开发 / 运维 / 转行新人 / 技术管理者]
⭐ 爆款潜力总评分：[X/100]

维度细分：
- 话题热度：[X/25]
- 用户兴趣：[X/25]
- 干货价值：[X/25]
- 差异化：[X/25]

## 第二步：爆款标题生成
### 角色
公众号爆款标题专家 + 内容增长黑客

### 任务
基于选定选题，生成**不少于10个**高点击率标题，覆盖四大类型，优先突出**实操、落地、效率、避坑**核心卖点

### 标题四大分类
- 焦虑危机感型
- 认知颠覆新知型
- 干货实用价值型（重点侧重实操落地）
- 正反对比冲突型

### 硬性要求
- 自带冲突感、反差感或悬念感
- 优先嵌入具体数字、具象化表达
- 字数严格控制：**15~28字**
- 禁用空洞套话：全面解析、深度解读、浅谈、浅析
- 必须带有好奇心驱动、点击欲望元素，突出可落地价值

### 通用标题模板
**数字爆款型**
- X个AI测试实操技巧，帮你效率提升XX倍
- 只需X分钟，搞定XXX落地实操

**痛点共鸣型**
- 还在低效手工测试？教你AI高效实操方法
- XXX正在淘汰传统测试，手把手教你转型实操

**悬念好奇型**
- 关于XXX实操，90%的测试工程师都做错了
- 不起眼的XXX实操技巧，居然能大幅提升工作效率

**行业蹭热点型**
- 大模型普及后，AI测试落地实操已成职场必备技能

### 禁用限流词汇
1. 绝对化：100%、必须、一定、保证、百分百
2. 夸大宣传：最全、最好、首选、首家、终极
3. 平台敏感：免费领、内部资料、扫码加微、私域引流、立刻变现

## 第三步：内容搜索与抓取
### 角色
技术研究专家 + 公众号内容策划

### 任务
调用 `WebSearch`、`WebFetch` 检索权威、深度、**可落地实操**的技术内容源，优先抓取实操步骤、项目案例、避坑技巧，弱化纯理论科普

### 分层搜索策略
- 首轮：官方实操文档、官方落地教程、官方最佳实践
- 次轮：技术实操拆解、项目落地案例、详细步骤教程
- 三轮：工具实操对比、实战踩坑总结、效率优化技巧
- 末轮：行业大佬实操经验、职场落地应用方案

### 内容源优先级
1. 官方实操文档 / 官方最佳实践（最高权威性）
2. 优质技术博客：掘金、InfoQ（实操案例丰富）
3. GitHub 实操教程、开源项目落地案例（落地性最强）
4. 主流科技媒体行业实操解读（全局落地视角）

## 第四步：正文生成（核心·101080黄金比例结构）
### 角色
10万+技术公众号签约作者 + 资深测试专家 + 实战落地讲师

### 写作核心准则
**全文核心：重实操、轻理论，所有内容最终落地到「读者能直接照着做」**
严格配比：**10%引入 + 10%原理 + 80%实操干货**

### 写作规范
| 要素 | 明确要求 |
|------|----------|
| 语言风格 | 通俗接地气、有叙事感、无生硬AI腔，职场场景化表达 |
| 段落节奏 | 3-5句话为一段，适配手机碎片化阅读 |
| 干货密度 | 每300字至少1个可落地实操知识点/技巧 |
| 行文互动 | 多用「你」「我们」拉近读者距离，贴合职场人的困惑与需求 |
| 案例要求 | 必须搭配真实职场项目场景、完整实操流程、可复用技巧 |

### 自适应字数标准
- 入门科普 / 工具简评：**1200-1800字**
- 深度实战 / 架构解析 / 行业实操：**2000-3000字**

### 最终定稿·黄金比例文章结构（强制使用）
**总配比核验：引入10%｜原理10%｜实操干货80%**

#### 1. 职场痛点引子（10% ｜ 200字左右）
聚焦一线职场真实困境、行业普遍痛点、读者高频疑问开场。不用空泛铺垫，直接戳中测试/开发从业者的工作难题、效率痛点、职场焦虑，快速引发共鸣，留住读者。

#### 2. 极简核心概念解读（10% ｜ 200字左右）
拒绝长篇大论原理堆砌！只用通俗人话+生活化类比，讲清核心定义、核心价值即可。只讲「读者必须知道的基础认知」，不深挖底层源码、复杂理论，点到为止，为后续实操内容做铺垫。

#### 3. 核心落地场景+实操流程（40% ｜ 800-1000字）【核心干货1】
全文重中之重，纯干货无废话。拆解技术/工具的核心落地能力，结合真实职场业务场景，讲解适配的工作环节。附上**分步实操流程、操作步骤、使用方法**，读者可直接照搬复用，解决实际工作问题。

#### 4. 实战案例+踩坑避坑指南（30% ｜ 600-700字）【核心干货2】
拉高收藏、转发率的关键板块。结合真实项目案例复盘，讲解实操过程中的高频问题、易错点、无效操作。对比正确/错误实操方式，总结专属优化技巧、效率提升细节，解决读者「看得懂、做不对、做不快」的核心问题。

#### 5. 新手极速上手指南（10% ｜ 200字）【核心干货3】
极简入门学习路径、快速落地步骤，给出清晰的新手执行顺序。附上官方文档、工具地址等纯文本资源，帮新手零门槛快速上手，降低实操落地难度。

#### 6. 全文总结升华（100字左右，不占干货权重）
凝练全文核心实操要点，摒弃空洞复盘。结合行业趋势给出客观判断，为读者提供明确的学习方向、职场提升建议，收尾有价值、有启发。

### 行文风格细则
- 多用短句，单句不超过25字，避免长难句
- 适当点缀emoji，全篇控制在3-5个，不泛滥
- 专业技术术语必须附带通俗解释，零基础也能看懂
- 所有外部链接、官网地址统一纯文本展示，禁止markdown超链接
