# 公众号多篇热点文章整合改写 Skill

## 概述

将用户提供的多篇同主题热点文章内容，经过自动去重、核心观点提炼、信息重组、句式重构后，整合改写为一篇合规原创、适合公众号发布的长文。产出内容可直接复制到公众号发布，规避查重，不抄袭原文。

## 触发条件

当用户表达以下意图时触发本 Skill：
- 提供多篇同主题文章要求整合/合并/改写
- 要求将多篇热点文章整合为一篇公众号文章
- 要求去重、提炼、重组多篇文章内容
- 要求将若干相关文章改写成一篇原创文章

## 输入参数接收

接收用户提供的以下参数，如用户未提供完整，逐一询问确认：

1. **多篇文章原文**（必需）—— 支持粘贴多篇内容，用户可用"文章1"/"文章2"或分隔线标记不同文章
2. **自定义文章主题**（可选）—— 用户可指定整合的主题方向或侧重点
3. **文风选择**（可选，默认"自媒体种草"）—— 可选值：严肃干货 / 大众科普 / 轻松口语 / 自媒体种草 / 职场深度
4. **输出字数**（可选，默认 1500-2000 字）—— 支持指定具体字数或范围
5. **是否需要公众号导语**（可选，默认需要）
6. **是否需要结尾互动话术**（可选，默认需要）

## 执行处理流程

严格按照以下步骤处理：

### 步骤一：拆解原文

分别对每一篇文章进行结构化拆解，提取以下要素：

| 要素 | 说明 |
|------|------|
| 事件背景 | 事件发生的起因、时间、地点、相关方 |
| 核心亮点 | 每篇文章最有价值的独特观点或信息增量 |
| 关键数据 | 数字、统计、比例、时间节点等事实数据 |
| 各方观点 | 不同立场、不同来源的观点和评论 |
| 冗余内容 | 重复段落、废话、广告性质内容、无关延伸 |

### 步骤二：自动去重

- 对比各篇文章中的相似段落和重复观点
- 剔除完全相同或高度相似的表述（保留第一次出现的版本或最优版本）
- 剔除废话水分、客套话、无信息量的过渡句
- 合并相同数据来源的重复数据

### 步骤三：结构重组

按照公众号爆款文章的标准结构重新搭建行文框架：

1. **爆款标题引入** — 用冲击力强、有信息量的开头抓住读者
2. **热点事件概述** — 简明扼要交代事件背景（1-2段）
3. **分模块深度解析** — 按主题维度拆分，每模块一个小标题，层层递进
4. **整体总结** — 提炼核心观点，升华主题
5. **结尾互动话术**（按需） — 引导点赞、在看、转发、评论区互动

### 步骤四：句式改写

对保留的所有信息进行改写，规则如下：

- **换语序**：主动改被动 / 主谓结构改宾语前置 / 调整因果顺序
- **换表达方式**：同义词替换、句式结构变换（陈述→转折→递进）
- **合并拆分**：长句合理拆分，短句适当合并
- **改写比例**：每句话必须经过改写，不得出现连续 3 句以上与原文句式结构相同
- **保留原则**：不得篡改热点中的真实时间、人物名称、具体数据、事件事实

### 步骤五：格式排版

- 正文自动合理分段（3-5 行一段）
- 模块间使用小标题分隔（## 级别）
- 关键数据建议用 **加粗** 突出
- 无多余符号、无错乱换行、无特殊乱码
- 全文无 Markdown 非必要格式（无需代码块包裹正文）

## 输出格式

严格按照以下顺序输出，**不得输出任何多余说明文字、思考过程或注释**：

```
### 备选标题

1. [标题1]
2. [标题2]
3. [标题3]

### 正文

[完整正文，分段清晰，可直接复制发布]
```

### 标题要求

- 3 个备选标题，风格需符合公众号爆款逻辑
- 覆盖不同角度（悬念型、数据型、观点型、共鸣型）
- 字数控制在 15-25 字之间
- 有吸引力、有信息量、不标题党

### 正文要求

- 字数严格控制在用户指定范围内
- 文风严格匹配用户选择的风格
- 逻辑流畅，段落间过渡自然
- 原创度 90% 以上，规避查重
- 可直接复制到微信公众号编辑器使用

## 约束规则

1. **严禁大段搬照原文**：必须整理解构后原创重组，连续相同表达不得超过 15 个字
2. **不得篡改事实**：热点时间、人物、数据、事件事实必须保留原貌
3. **只输出成品**：不输出额外解释、思考过程、闲聊、建议
4. **严格遵循参数**：文风、字数、是否包含导语/互动话术，必须遵守用户指定
5. **回避风险**：涉及争议性内容以事实陈述为主，不主观站队
